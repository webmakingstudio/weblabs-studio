🔴 Seguridad — Acciones urgentes

4

Estos 4 ítems deben resolverse antes de hacer público el repositorio o desplegar en producción.

🔐 1. Revocar API key de Resend

.env.local La key real fue commiteada. Ve a dashboard.resend.com → API Keys → revocar → generar nueva. Copia la nueva key a .env.local.

🧹 2. Limpiar .env.local del historial de git

.env.local git rm --cached .env.local && git commit como mínimo. Si ya has hecho push, usa git filter-branch o BFG Repo-Cleaner. Después, verifica que .gitignore lo excluye (ya está incluido).

👤 3. Datos personales en el repositorio

legal.js Nombre completo, NIF, dirección y teléfono están en texto plano en las traducciones. Si el repo es o será público, valora mover estos datos a variables de entorno o a un CMS.

🟡 4. Email y teléfono visibles en el código

script.jsindex.html El email y teléfono están en el HTML/JS. Es normal en una landing, pero atraerá spam. El honeypot del formulario ayuda, pero no hay protección anti-scraping en los mailto: y enlaces visibles.
